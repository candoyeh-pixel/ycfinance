# 奕成財創｜商業計畫書（月報服務模式）

> **一句話定位**：成為台灣中小企業的外包財務長——用月報服務收集數據，用產業 Benchmark 建立護城河，用金融媒合放大營收。

---

## 一、市場定位

### 市場缺口

| | 稅務報表 | 管理報表 |
|---|---|---|
| 誰負責 | 會計師事務所 | **通常沒有人** |
| 目的 | 報稅合規 | 幫老闆做決策 |
| 老闆能用嗎 | 很難直接用 | **可以直接用** |

記帳士做合規，但「把數字翻譯成老闆能用的決策資訊」這件事，沒有人在做。

### 為什麼是真缺口

- 記帳士的商業模式靠報稅量，做管理報表不划算
- 會計師事務所嫌 SME 客單價太低
- ERP 軟體裝了但老闆不會用
- **不是沒有人做，是做了活不下去的居多**

### 缺口裡的三個風險

1. **老闆的付費意願**：覺得已經付了記帳費，管理報表是「加值服務」不是必需品
2. **數據品質問題**：底層帳是亂的，報表會假精準
3. **消滅信使效應**：報表顯示壞消息，老闆質疑方法而不改決策

**對策**：從「已經在痛」的客戶切入，不說服還沒感覺痛的人。

### 「已經在痛」的場景

| 場景 | 為什麼容易成交 | 找到的方法 |
|---|---|---|
| 剛換會計 / 財務人員 | 交接期帳務最亂，老闆最焦慮 | 每天掃 104 新刊職缺 |
| 準備融資但帳很亂 | 投資人要數字，才發現拿不出手 | 新創社群、天使投資人轉介 |
| 被稅務稽查 / 年底稅務壓力 | 問題已爆發，付費意願高 | 報稅季前主動接觸；會計師轉介 |
| 要賣公司 / 引進股東 | Due diligence 需要乾淨財報 | M&A 顧問、銀行 RM 轉介 |

## 二、核心戰略洞察

### 你不是在做事務所，你在建一間數據公司

月報是收集數據的手段，不是生意本身。

```
                    ┌─────────────────┐
                    │  產業 Benchmark  │ ← 這是護城河
                    │ （跨客戶數據）    │
                    └────────┬────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
    ┌─────┴─────┐    ┌─────┴──────┐    ┌─────┴──────┐
    │ 月報服務   │    │ 診斷 + 預警 │    │ 金融媒合   │
    │ Per Node  │    │ 可規模化    │    │ 交易抽佣   │
    │ 穩定現金流 │    │ 真正的產品  │    │ 高利潤     │
    └───────────┘    └────────────┘    └────────────┘
     養數據的入口       真正的產品         變現的引擎
```

**當你服務同產業 30 家客戶，你掌握的 Benchmark 數據是記帳士沒有、會計師沒有、任何單一公司財務長也沒有的。這才是真正的護城河。**

## 三、商業模式

### 服務交付模式

```
Cando 定規格 → 內部小工具輔助 → 財務人員（IVY）執行 → 客戶每月收到月報
                                                      （偶爾加顧問會議）
```

- **客戶不碰系統**，只收到月報（他們不想自己用 SaaS）
- 財務人員用內部工具提高效率
- Cando 負責定產業標準、品質把關、談合作夥伴

### 三層收入結構

| 層級 | 價值 | 收費方式 | 毛利 | 可規模化 |
|------|------|---------|------|---------|
| **數據層**（月報） | 每月經營診斷 | Base + Per Node | 70-80% | 線性（靠人） |
| **洞察層**（benchmark） | 產業比較、異常預警 | 固定月費 | 90% | 高（靠軟體） |
| **行動層**（金融媒合） | 融資轉介、保險、節稅 | 成交抽佣 | 95% | 爆發力強 |

## 四、目標客群

### 甜蜜點：卡在中間的成長型企業

> 剛過了「老闆一個人能搞定」的階段，但還沒到「請得起財務長」的規模。

| | 不要碰 | 甜蜜點 | 太大吃不下 |
|---|--------|--------|-----------|
| 餐飲 | 單店、小吃店 | **3-20 家連鎖** | 50 家以上集團 |
| 電商 | 一人蝦皮賣家 | **年營收 3,000 萬 - 3 億品牌** | 大型平台 |

### 為什麼是這個區間

**餐飲 3 家連鎖以上**：
- 老闆開始看不清每家店的狀況
- 食材統一採購，成本分攤變複雜
- 展下一家店，銀行要看財務報表

**電商年營收 3,000 萬以上**：
- 多通路（官網 + 蝦皮 + momo），帳混在一起
- 有庫存，算毛利很痛苦
- 5-10 個員工，人事成本突然變重要

### 台灣市場規模

| 產業 | 符合條件估計數量 |
|------|-----------------|
| 連鎖餐飲（3-20 店） | 2,000 - 3,000 家 |
| 中型電商品牌 | 3,000 - 5,000 家 |
| **合計** | **約 5,000 - 8,000 家** |

目標 120 家 = 市場滲透率不到 2%。市場夠大。

### 產業選擇建議

首選 **餐飲連鎖**：
- POS 資料結構化程度高，資料最好接
- 關鍵指標全產業通用：食材成本率、人事佔比、翻桌率、坪效
- 餐飲老闆圈子緊，口碑傳播快
- 殺手級場景：「你的 5 家店裡，這家本月虧 8 萬，食材成本比其他店高 6%」

備選 **診所/醫美**：
- 客單價高，付 5 萬月費不猶豫
- 醫生很會賺但完全不懂財務
- 但需要人脈帶進去，信任感建立慢

**實際決策：看 Cando 手上 Lance、NTU 等現有客戶最集中在哪個產業，就從那裡深耕。不要從零開始選。**

## 五、定價策略

> 完整定價細節、議價邊界、話術請見 👉 [定價策略｜完整版 2026.md](定價策略｜完整版%202026.md)

### 定價摘要：Base + Per Node

```
Phase 0 快篩（18,000 / 28,000）→ 抵扣首月月費
  → 導入費（30,000 / 50,000 一次性）
    → 月費 = 基礎費 + Node × 單價 + 加掛模組
      → 金融媒合佣金（行動層收入）
```

**月費公式**：

| 項目 | 費用 |
|---|---|
| 基礎費（三層） | 年營收 < 1 億：20,000 / 1-3 億：25,000 / 3 億+：30,000 |
| 標準 Node | +5,000/個（一般門市、通路、產品線） |
| 複雜 Node | +8,000/個（有獨立庫存、獨立採購、或跨境） |
| 加掛：現金流管理 | +10,000 |
| 加掛：預算管理 | +10,000 |
| 加掛：月度策略會議 | +15,000 |

- **基礎費**三層，依營收規模分，簽約時定一次不再調整
- **Node** = 需要獨立追蹤損益的最小單位（門市、通路、院所、產品線）
- **超大客戶**（年營收 5 億+）改用營收 × 0.01%/月，月費上限 15 萬

**各產業月費範例**：

| 客戶 | 層級 | 月費 |
|---|---|---|
| 小型服務業 1 node（年營收 3,000 萬） | 一般 | 25,000 |
| 餐飲 3 家店（年營收 5,000 萬） | 一般 | 35,000 |
| 電商 3 通路 + 庫存（年營收 5,000 萬） | 一般 | 43,000 |
| 貿易公司 1 node（年營收 3 億） | 大型 | 35,000 |
| 餐飲 10 家店（年營收 1.5 億） | 中大型 | 75,000 |
| 餐飲 20 家店（年營收 4 億） | 大型 | 130,000 |

**關鍵設計**：
- Per Node 讓月費跟客戶事業規模掛勾，客戶開新店你自動加收
- 營收 % 當話術（「不到你營收的 1%」）讓老闆覺得便宜
- 半年約 9 折綁定（前 2 個月還在熟悉帳務，月繳月退客戶第二個月就跑）
- 對外統一叫 **「每月經營診斷」**，不叫月報服務

## 六、服務設計

### 月報要有「老闆頁」

整份報表 5-10 頁，但第一頁只放三個數字：
1. 這個月賺多少
2. 帳上還有多少現金
3. 跟上個月比是變好還是變差

老闆只看這頁。其他是給幹部或股東看的。

### 交付 SOP

1. 財務人員收集實體單據、上傳
2. 內部工具自動分類、對科目
3. 財務人員檢查、修正例外
4. 系統產出月報
5. LINE 群發送月報 + 30 秒語音/三行文字摘要

### LINE 群 SOP

台灣老闆不看 email、不開系統。月報做好丟 LINE 群，附文字重點摘要。成本極低但客戶感受差很多。

## 七、團隊與組織

### 財務人員策略

- **不要找太資深的**，找會計系畢業 1-3 年、細心、願意照 SOP 做事的人
- 資深的人有自己的做法，反而難標準化
- 一個財務人員平均服務 **10 家客戶**（有工具輔助後可到 15-20 家）

### Cando 的角色

**80% 時間花在**：定產業標準、建 Benchmark 模型、談金融合作夥伴、品質把關
**20% 時間花在**：帶新人、處理重大例外

如果 Cando 一直自己做月報，公司永遠是事務所。

### 人事成本結構

| 職位 | 月薪（含勞健保） | 人均服務客戶數 | 人均月產出 |
|------|-----------------|--------------|-----------|
| 財務人員 | 60,000 - 70,000 | 10 家 | 300,000 |
| 資深顧問 | 80,000 - 100,000 | 5 家（高價） | 500,000 |

## 八、軟體發展策略

### 不是做 SaaS，是分階段提高人效

```
現在      → 內部工具給財務人員用，提高人效
6-12 個月  → 客戶端唯讀 Dashboard，提高體驗感受
2 年後    → 看數據量決定要不要做 SaaS（可能永遠不需要）
```

### 第一階段：內部工具

```
財務人員上傳資料 → 系統自動分類、對科目 → 人工檢查修正 → 月報自動產出
```

目的：一個人從服務 10 家變成服務 20 家。客戶完全不碰系統。

**MVP 開發估算**：

| 項目 | 工具 | 時間 |
|------|------|------|
| Excel 上傳解析 | Python | 1 週 |
| 標準化科目對應 | 規則引擎 + 人工修正 | 2 週 |
| 報表產出 | Dashboard 工具 | 1 週 |
| AI 文字摘要 | Claude API | 幾天 |
| 簡單前端 | 任何框架 | 2 週 |

**一個工程師 + 財務人員，6-8 週可以出 MVP。**

### 第二階段：客戶唯讀 Dashboard

客戶隨時能看，不用等月底。但客戶不操作，只是看。

### 第三階段：產業 Benchmark 系統

同產業 30 家以上客戶數據 → 自動產出產業比較報告。這是產品，不是服務。

### 關鍵原則

**從第一天起就要把客戶的財務數據結構化存好。** 這些數據是之後做 Benchmark、做產品的原料。散在 Excel 裡以後要整理會非常痛。

### 轉型時機判斷

| 訊號 | 代表什麼 |
|------|---------|
| 同產業客戶超過 20-30 家 | Benchmark 數據有統計意義 |
| 月報流程已經 80% 固定 | 可以寫成軟體 |
| 客戶開始問「我跟別人比怎麼樣」 | 市場在要求產品 |

三個訊號出現兩個，就該投資開發系統。

## 十一、其他收入機會

### 稅務籌劃（不是報稅）

記帳士做合規，但「怎麼合法少繳稅」沒人主動幫老闆想。直接省錢，付費意願高。

### 財務健診 + 轉介網

幫老闆做完財務分析，發現需要保險、貸款、法律諮詢 → 轉介專業人士收介紹費。你變成老闆的「財務窗口」，背後是一張服務網。

### 接班/股權規劃

台灣 SME 老闆老化嚴重，不知道怎麼傳承。高單價、低競爭、老闆願意花錢。

## 十二、營收模型｜到 1 億的路徑

### Per Node 模式的威力：客戶不增加，營收也會成長

傳統固定月費模式，客戶數不變 = 營收不變。
Per Node 模式，客戶開新店、加新通路，你的營收自動成長。

**範例**：一家餐飲客戶從 3 家店擴到 8 家店
- 入會時月費：15,000 + 3×6,000 = 33,000
- 兩年後月費：15,000 + 5×6,000 + 3×5,000 = 60,000
- **同一個客戶，營收成長 82%，你沒多做任何銷售。**

### 120 家客戶版（務實版）

| 客戶類型 | 家數 | 平均 node 數 | 平均月費 | 月營收 |
|---------|------|-------------|---------|--------|
| 小型（2-3 nodes） | 40 家 | 2.5 | 30,000 | 120 萬 |
| 中型（5-8 nodes） | 55 家 | 6 | 52,000 | 286 萬 |
| 大型（12-20 nodes） | 25 家 | 15 | 95,000 | 237.5 萬 |
| **月費小計** | **120 家** | | | **643 萬** |

| 其他收入 | 算法 | 月營收 |
|---------|------|--------|
| 導入費 | 每月新增 3 家 × 40,000 | 12 萬 |
| 年度結算（攤月均） | 80 家 × 50,000 ÷ 12 | 33 萬 |
| 金融媒合 | 月均 5 案 × 100,000 | 50 萬 |
| 顧問 + 專案 | | 60 萬 |
| **其他小計** | | **155 萬** |

| | **月營收合計** | **年營收** |
|---|---|---|
| **Total** | **~800 萬** | **≈ 9,600 萬** |

> 120 家客戶用 Per Node 模式就能逼近 1 億，因為大型客戶的月費被 node 數自然拉高。加上客戶每年平均成長 1-2 個 node，同一批客戶的營收年增約 10-15%。

### 破億的關鍵不是多拿客戶，是讓現有客戶長大

| 年份 | 客戶數 | 平均 node 數 | 平均月費 | 月營收（含其他） | 年營收 |
|------|--------|-------------|---------|----------------|--------|
| Y1 | 10 | 3.5 | 36,000 | 50 萬 | 600 萬 |
| Y2 | 35 | 4.5 | 42,000 | 180 萬 | 2,160 萬 |
| Y3 | 70 | 5.5 | 48,000 | 430 萬 | 5,160 萬 |
| Y4 | 120 | 6.5 | 54,000 | 850 萬 | **1 億** |

> Y3 到 Y4 的跳躍：客戶數從 70 → 120（+70%），但平均月費從 48K → 54K（+12%）因為既有客戶在擴店。Per Node 讓你兩邊同時成長。

### 團隊結構（Y4 破億版）

| 角色 | 人數 | 職責 |
|------|------|------|
| Cando（創辦人） | 1 | 產業標準、品控、合作夥伴 |
| 資深顧問 | 3-4 | 大型客戶、品質覆核、策略會議 |
| 財務人員 | 15-18 | 月報產出執行（人均服務 7-8 家） |
| 業務/行銷 | 1-2 | 獲客、社群經營 |
| **合計** | **~22-25 人** | |

> Node 多的大型客戶工時較高，但單位 node 的工時會下降（同一家公司的店，資料格式一樣）。人均服務客戶數會比純算家數更低，但人均產出金額更高。

## 十三、4 年路線圖

### Year 1：驗證（目標 600 萬）

**核心任務：把流程從 Cando 的腦袋變成 SOP**

| 行動 | 目標 |
|------|------|
| 穩定服務 10 家客戶（含 Lance、NTU） | 月營收 30 萬 |
| 確認月報格式、工時、定價 | 可複製的 SOP v1 |
| 選定一個產業深耕 | 產業案例開始累積 |
| 招第 1 個財務人員 | Cando 開始脫離執行 |
| 開發內部工具 MVP | 人效提升 50% |

**KPI**：10 家客戶、1 個財務人員、月報 SOP 定稿

### Year 2：複製（目標 2,000 萬）

**核心任務：證明「不是只有 Cando 能做」**

| 行動 | 目標 |
|------|------|
| 招到 3-5 個財務人員 | 新人 2 週內能獨立出月報 |
| 客戶數到 40-50 家 | 月營收 150-170 萬 |
| 建立品質覆核機制 | 資深人員抽查，不是每份都看 |
| 開始跟記帳士合作轉介 | 被動線索來源建立 |

**KPI**：50 家客戶、5 個財務人員、新人 onboarding ≤ 2 週

### Year 3：加速（目標 5,000 萬）

**核心任務：洞察層開始產生營收**

| 行動 | 目標 |
|------|------|
| 同產業客戶超過 30 家 | Benchmark 數據有意義 |
| 推出產業比較報告 | 洞察層開始收費 |
| 團隊 10-15 人 | 人均服務量拉到 15 家 |
| 開始金融媒合 | 行動層營收啟動 |
| 開發客戶端唯讀 Dashboard | 客戶體驗升級 |

**KPI**：80 家客戶、Benchmark 產品上線、金融媒合首單

### Year 4：破億（目標 1 億）

**核心任務：Per Node 飛輪全速運轉**

| 行動 | 目標 |
|------|------|
| 120 家客戶 | 新客持續進，舊客持續擴 |
| 平均 node 數成長到 6.5 | 既有客戶的自然擴張 |
| 產業 Benchmark 成為標配 | 加掛模組採用率 > 50% |
| 金融媒合常態化 | 每月穩定 5 案以上 |
| 團隊 ~22-25 人 | 組織穩定運作 |

**KPI**：月營收 850 萬、平均月費 54,000、客戶 node 年增率 > 10%

## 十四、關鍵風險與對策

| 風險 | 嚴重度 | 對策 |
|------|--------|------|
| 人的品質（月報出錯一次客戶就跑） | 高 | 標準化 SOP + 資深人員抽查覆核 |
| 客戶流失（看幾個月覺得還好） | 高 | 月報裡放「嚇一跳指標」、綁半年約 |
| 業務瓶頸（120 家 + 補流失） | 高 | 記帳士合作分潤、iCHEF 等平台合作 |
| 數據品質（帳是亂的，報表假精準） | 中 | 導入期先整帳，月報標注數據可信度 |
| 低價競品 | 中 | 用 Benchmark 差異化，這個別人沒有 |
| Cando 時間瓶頸 | 中 | 盡早脫離執行，專注規格和品控 |
