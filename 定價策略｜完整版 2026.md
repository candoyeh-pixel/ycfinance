# 定價策略｜完整版 2026（月報服務模式）

> 本文件是定價的唯一來源。商業計畫書和提案簡報引用本文件，不重複定價細節。

---

## 一、定價原則

1. **Per Node 計價**：月費跟客戶的事業規模掛勾（門市數、通路數、帳套數），客戶成長你自動加錢
2. **從結果定價，不從工時定價**：賣「每月經營診斷」，不賣時間
3. **Phase 0 是入口，不是免費諮詢**：18,000 傳遞高價值訊號，且可抵扣首月月費
4. **半年約鎖定黏性**：保護現金流，降低流失率
5. **營收 % 當話術，不當定價**：「只佔你營收不到 0.5%」讓老闆覺得便宜
6. **不要叫月報服務**：對外統一叫「每月經營診斷」

---

## 二、核心定價模型：Base + Per Node

```
月費 = 基礎費（彙總報表 + 現金流預測 + 老闆頁 + 會議）
     + 每個 Node × 單價（每個 Node 的獨立損益 + 同組比較）
     + 加掛模組（可選）
```

### 什麼是 Node

Node = 客戶事業裡需要獨立追蹤損益的最小單位。

| 產業 | Node 定義 | 判斷標準 |
|---|---|---|
| 餐飲連鎖 | 門市 | 有獨立 POS 的營業據點（中央廚房不算） |
| 電商品牌 | 銷售通路 | 有獨立營收的通路（官網、蝦皮、momo 各算一個） |
| 診所/醫美 | 院所 | 有獨立掛號/收費系統的據點 |
| 製造代工 | 產品線 or 主要客戶 | 需要獨立計算毛利的單位 |
| 連鎖零售 | 門市 | 同餐飲 |
| 多公司集團 | 帳套/公司 | 獨立的法人實體 |

> ⚠️ Node 的定義必須寫進合約，避免後續爭議（例：外帶店算不算一個 node？加盟店算不算？）

## 三、定價表

### Phase 0：財務快篩（獲客入口）

| 方案 | 定價 | 包含 |
|---|---|---|
| **快速健診** | **NT$ 18,000** | 2 小時線上，四大面向風險快篩，口頭報告 + 1 頁摘要 |
| **完整健診** | **NT$ 28,000** | 3～4 小時含帳務文件審閱，書面報告（Top 5 風險 + 30 天行動清單） |

> **健診後 30 天內簽約月費，健診費全額抵扣首月**——讓 Phase 0 成為月費的試用金，大幅提高轉換率。

### 導入期（一次性）

導入期重點：了解客戶帳務、建立月報模板、對齊科目、定義 Node。

| 方案 | 定價 | 週期 | 適用情況 |
|---|---|---|---|
| **標準導入** | **NT$ 30,000** | 2 週 | 帳務基本可用，對科目 + 建模板 |
| **完整導入** | **NT$ 50,000** | 4 週 | 帳務較亂，需整理歷史帳 + 建立口徑規範 |

> 導入方案依 Phase 0 健診結果決定，非客戶自選。

---

### 月度經營診斷（核心收入）— 複雜度層級 + Per Node

#### 基礎費（依複雜度層級，每個客戶都收）

帳務複雜度通常跟營收規模正相關：交易量越大、供應商越多、帳務結構越複雜。用營收區間做分層，但對客戶講的是「複雜度」不是「營收」。

| 層級 | 年營收區間 | 基礎費（月付） | 基礎費（半年約 9 折） | 典型客戶 | 痛點話術 |
|---|---|---|---|---|---|
| **一般** | 1 億以下 | **NT$ 20,000** | NT$ 18,000 | 3-5 家連鎖餐飲、中型電商、單店服務業 | 「你的營收在成長，但毛利為什麼沒跟著成長？」 |
| **中大型** | 1 億～3 億 | **NT$ 25,000** | NT$ 22,500 | 10 家連鎖、貿易公司、中型製造 | 「你的報表能直接拿給投資人或董事會講嗎？」 |
| **大型** | 3 億以上 | **NT$ 30,000** | NT$ 27,000 | 大型連鎖、大型製造、多公司集團 | 「你需要的不是記帳，是一整個財務團隊。」 |

> **甜蜜點是一般層（1 億以下）**：台灣大多數成長中的 SME 落在這裡。複雜度在掌控範圍內，支付能力充足，轉介效率高。

#### 層級是內部工具，客戶永遠看不到

- 層級只存在於 Cando 的報價計算器裡，**客戶不知道有層級這件事**
- 客戶看到的永遠只是一個報價數字（例如「月費 45,000」），不是公式
- 簽約時 Cando 依營收和複雜度判定層級、算出報價，報一個數字
- 合約裡只寫月費金額和 per node 加價規則，不寫層級

#### 月費什麼時候會變？

**會變的**（客戶能理解且接受）：
- 開新店 / 加新通路 → node 增加，每個 +5,000-6,000
- 客戶主動要求加掛模組（現金流、策略會議等）
- 工作範圍有實質變更（例如開始做跨境、多了子公司）→ 以新增模組或 node 的方式處理

**不會變的**（避免信任流失）：
- 客戶營收成長但 node 數不變 → **不加價**（同店成長是客戶的本事）
- 不會有「你的層級升級了所以基礎費調整」的對話

**加價的理由永遠是「工作範圍變了」，不是「你賺更多了」。**

> 如果 Cando 初期低估了客戶的複雜度導致虧工時，就當學費——下一個同規模客戶報準就好。不要回頭跟既有客戶補收差價。

> 對客戶的說法：「基礎費是看帳務複雜度——交易量、供應商數量、帳務結構。規模越大通常越複雜，所以會落在比較高的層級。」
> **不要說「因為你營收高所以比較貴」。**

基礎費包含：
- 彙總損益表（全公司）
- 現金流預測（未來 3 個月）
- 老闆頁（30 秒看完的摘要）
- LINE 群月報推送
- 每月 1 次 30 分鐘電話

#### Per Node 費用

| Node 類型 | 單價/月 | 判斷標準 |
|---|---|---|
| **標準 Node** | **NT$ 5,000** | 一般門市、一般通路、一般產品線 |
| **複雜 Node** | **NT$ 8,000** | 有獨立庫存管理、獨立採購流程、或涉及跨境 |

每個 Node 包含：
- 該 Node 的獨立損益表
- 跟其他 Node 的比較分析（同店比較、同通路比較）
- 異常警示（成本率偏高、毛利下滑等）

> Node 數量超過 20 個 → 考慮改談固定總價（月費上限約 15 萬）

#### 加掛模組（可選）

| 模組 | 月費 | 內容 |
|---|---|---|
| **現金流管理** | +10,000 | 未來 3 個月收支預測 + 缺口預警 |
| **預算管理** | +10,000 | 年度預算 + 每月差異追蹤 |
| **月度策略會議** | +15,000 | 升級為每月 1 次 60 分鐘深度會議 + 行動清單 + 每季檢視 |
| **庫存管理** | +10,000 | 庫存周轉分析 + 呆滯預警（電商、零售適用） |

## 四、定價心理學設計

### 為什麼 Per Node 比固定方案好賣

| 固定三方案 | Per Node |
|---|---|
| 「為什麼標準版比基礎版貴？」 | 「因為你多 2 家店」——客戶秒懂 |
| 客戶開新店，你要重新談價 | 自動加錢，合約裡就寫好了 |
| 客戶關店，你不好意思不降價 | 自然減少，客戶覺得公平 |
| 大客戶小客戶同價 | 自然分層，大客戶付更多 |

### 錨定效應：先算大的再算小的

提案時先幫客戶算「如果你有 10 家店」的價格，再算「你現在 5 家店」：
```
10 家店：70,000/月 → 「但你現在 5 家，只要 45,000」
```
客戶感覺自己撿到便宜，而且看到未來的價格也不會嚇到（因為到時候營收也更高了）。

### 營收 % 話術

Per Node 定價，但談判時用營收 % 降低客戶的價格感知：

| 客戶 | 層級 | 月費 | 年營收 | 佔比 | 話術 |
|---|---|---|---|---|---|
| 3 家餐飲 | 一般 | 35,000 | 5,000 萬 | 0.8% | 「不到你營收的 1%」 |
| 5 家餐飲 | 一般 | 45,000 | 8,000 萬 | 0.7% | 「不到你營收的 1%」 |
| 10 家餐飲 | 中大型 | 75,000 | 1.5 億 | 0.6% | 「只佔你營收的 0.6%」 |
| 貿易公司 1 node | 大型 | 35,000 | 3 億 | 0.1% | 「只佔你營收的 0.1%」 |
| 4 通路電商+庫存 | 中大型 | 68,000 | 1.2 億 | 0.7% | 「不到你營收的 1%」 |

### Phase 0 抵扣設計

```
客戶心理流程：
「先花 18,000 試試看」
→ 健診後看到具體問題
→ 「反正 18,000 可以抵月費」
→ 簽月費方案
```

## 五、議價邊界（對外不說，自己要清楚）

折扣只換條件（半年約 / 預付 / 介紹新客戶），不接受「就是覺得貴」的砍價。

| 項目 | 標準價 | 底線 | 可給折扣的條件 |
|---|---|---|---|
| Phase 0 完整版 | 28,000 | 22,000 | 同時確認後續合作意願 |
| 導入費（完整） | 50,000 | 38,000 | 搭配半年約一起簽 |
| 基礎費（一般） | 20,000/月 | 18,000/月 | 半年約 |
| 基礎費（中大型） | 25,000/月 | 22,500/月 | 半年約 |
| 基礎費（大型） | 30,000/月 | 27,000/月 | 半年約 |
| 標準 Node | 5,000/node | 4,500/node | 半年約 + 5 個 node 以上 |
| 複雜 Node | 8,000/node | 7,000/node | 半年約 |
| 加掛模組 | 各模組標準價 | 9 折 | 加掛 2 個以上模組 |

**底線規則**：
- 任何客戶月費不低於 **22,000**（低於這個數成本 cover 不住）
- 半年約的折扣上限是 **10%**，不能再多
- 月費超過 **150,000** → 改談固定總價

## 六、Node 上限與大客戶處理

| Node 數量 | 處理方式 |
|---|---|
| 1-15 個 | 標準 Per Node 計價 |
| 16-30 個 | Per Node 計價，但考慮固定總價 |
| 30 個以上 | **改談固定月費**（通常 12-18 萬），不再按 node 算 |

> 超過 30 個 node 的客戶，按 node 算會超過 15 萬/月。這個價格區間的客戶可能會考慮請全職財務主管。改談固定總價，控制在 12-18 萬，強調「一整個團隊 vs 一個人」的差異。

## 七、客戶比較基準管理

老闆會拿你跟這些比，要在提案前就把認知差距拉開：

| 比較對象 | 費用 | 你的應對 |
|---|---|---|
| 記帳行 | 3,000～8,000/月 | 「他們記錄發生的事，我讓你用這些數字做決策。」 |
| 事務所月顧問 | 8,000～20,000/月 | 「他們做合規，我做經營診斷。」 |
| 自己雇會計 | 35,000～45,000/月含勞健保 | 「差不多的價格，但你得到的是一整個財務團隊，不請假不離職。」 |
| 財務經理 | 55,000～80,000/月 | 「比請人便宜，而且不用管人事。我們還能告訴你跟同業比怎麼樣。」 |

## 八、對客戶常見反應的應對話術

**「太貴了，記帳事務所只收 5,000 一個月。」**
> 「記帳是記錄已發生的事，我做的是讓你看懂這些數字、用它來做決策。
> 你現在每個月拿到的報表，你能直接用來決定明天要做什麼嗎？」

**「可以先免費試用嗎？」**
> 「我們有財務快篩，18,000 元，兩小時你就能知道目前最該先處理的三個問題。
> 如果之後你決定繼續，這 18,000 直接抵第一個月費用。」

**「先做三個月試試？」**
> 「可以，但月付方案價格會高一點。如果你確定要做，半年約每個月省 10%。」

**「我只有一家店，需要嗎？」**
> 「一家店的基礎方案是 21,000/月。如果你計劃展店，現在把財務基礎建好，
> 開第二家的時候你就不會像很多老闆一樣——等到虧了才發現。」

**「以後開新店，費用會一直加上去嗎？」**
> 「對，每家新店加 5,000-6,000/月。但你想——開新店代表營收在成長，
> 多花幾千塊就能馬上知道新店賺不賺錢，比開了半年才發現虧錢便宜太多了。」

**「營收不穩定，怕付不起。」**
> 「這個月費大概佔你營收不到 1%。而且如果你關了一家店，那個 node 的費用也會拿掉，
> 我們的收費跟你的規模是同步的。」

## 九、漲價策略

- 前 10-20 家客戶用現有 node 單價，換取口碑和案例
- 有了標準化模板 + 客戶見證後，**第 21 家開始調高 node 單價**（6,000 → 7,000）
- 舊客戶的 node 單價不動，但**新增的 node 用新價格**
- 加掛模組的價格可以隨時調，因為是新增服務

| Phase 0 健診 | 25 次 × 23,000 均 | 575,000 |
| 導入費 | 15 案 × 40,000 均 | 600,000 |
| 月費 | 15 家 × 平均 45,000 × 8 個月 | 5,400,000 |
| 年度結算 | 8 家 × 50,000 | 400,000 |
| 金融媒合 | 5 案 × 100,000 均 | 500,000 |
| **合計** | | **≈ 750 萬** |

> 平均月費拉高到 45,000 是因為部分客戶在年中擴店，node 數自然增加。這就是 Per Node 模式的威力。

標準公式不適用時，改用營收比例：

```
月費 = 年營收 × 0.01%（月）+ 複雜度加成
```

| 年營收 | 基礎月費（0.01%） | 複雜度加成範例 | 月費 |
|--------|-----------------|-------------|------|

| 20 億+ | 面議 | — | 固定總價上限 15 萬 |

> 月費超過 15 萬 → 改談固定總價。超過這個區間客戶會考慮請全職財務長。
